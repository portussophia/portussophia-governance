# Document Review Workflow

## Purpose

Ensure all governance documents meet quality, accuracy, and compliance standards before approval.

## Scope

This workflow applies to:

- New governance documents
- Significant updates to existing documents
- Changes to sealed artifacts

## Process Steps

### 1. Preparation

**Contributor Actions:**
- Create document with valid UICH header
- Follow applicable standards (naming, paths, metadata)
- Include clear, neutral content
- Self-review using compliance checklist
- Submit via pull request

**Prerequisites:**
- UICH header complete and valid
- Appropriate file location and naming
- Proper trademark usage (PortusSophia™)
- Neutral, governance-appropriate tone

### 2. Initial Review

**Steward Actions:**
- Verify UICH compliance
- Check path and naming conventions
- Confirm appropriate scope (governance-only)
- Review for trademark accuracy
- Assess tone and clarity

**Review Criteria:**
- ✅ Valid UICH header
- ✅ Correct location and filename
- ✅ Governance-appropriate content
- ✅ Proper PortusSophia™ usage
- ✅ Neutral, technical tone
- ✅ No canon/narrative content
- ✅ Clear, accurate information

### 3. Feedback and Revision

**Process:**
- Steward provides specific feedback
- Contributor addresses comments
- Iterative refinement as needed
- Re-review after significant changes

**Common Issues:**
- Missing or invalid UICH fields
- Incorrect naming or paths
- Trademark format errors
- Non-neutral tone
- Out-of-scope content
- Unclear or ambiguous language

### 4. Approval

**Criteria for Approval:**
- All review criteria met
- No unresolved feedback
- Compliance checklist complete
- Steward approval granted

**Steward Actions:**
- Mark as approved
- Update document status to "approved" or "review"
- Merge pull request
- Update related documentation if needed

### 5. Post-Approval

**Actions:**
- Document status updated in UICH
- Related documents cross-referenced if needed
- Announcement (if significant)
- Monitoring for issues or questions

## Review Timeline

- **Initial Review**: Within 5 business days
- **Feedback Response**: As soon as practical
- **Final Approval**: Within 2 business days after addressing feedback

Expedited review available for critical updates.

## Sealed Artifact Review

Documents with `status: sealed` require enhanced review:

1. Issue discussion before PR submission
2. Governance team consensus
3. Impact assessment
4. Migration plan (if breaking changes)
5. Extended review period (10 business days)

## Roles

### Contributor
- Prepares document
- Addresses feedback
- Ensures compliance

### Steward
- Reviews submission
- Provides feedback
- Approves or requests changes

### Governance Team
- Final authority on sealed artifacts
- Resolves disputes
- Maintains process
