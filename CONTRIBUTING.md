# Contributing to PortusSophia™ Governance

## Governance-Only Policy

This repository maintains **strict governance discipline**. All contributions must align with governance purposes only.

## Before Contributing

1. Verify your contribution is governance-related
2. Check that similar standards don't already exist
3. Ensure your contribution maintains neutral tone
4. Confirm you're not introducing canon or narrative content

## Contribution Types

### ✅ Acceptable Contributions

- Improvements to UICH specifications
- Metadata schema enhancements
- Path and naming convention clarifications
- Stewardship workflow refinements
- Compliance checklist updates
- Documentation fixes (grammar, clarity)
- Trademark usage corrections

### ❌ Unacceptable Contributions

- Canon storylines or narrative content
- Character profiles or worldbuilding
- Public-facing marketing material
- WebKernel implementation code
- Creative or fictional content
- Speculative features or inventions
- Modifications to sealed governance artifacts

## Contribution Process

1. **Fork** this repository
2. **Create** a branch with a descriptive name (e.g., `fix/uich-template-typo`)
3. **Make** minimal, focused changes
4. **Test** that your changes don't break existing standards
5. **Submit** a pull request with clear description
6. **Respond** to review feedback

## Style Guidelines

### Tone
- Neutral and technical
- Objective and factual
- Clear and precise
- No promotional or creative language

### Formatting
- Use Markdown for documentation
- Include YAML frontmatter where applicable
- Follow existing document structure
- Maintain consistent indentation (2 spaces for YAML, 4 for code blocks)

### Terminology
- Use **PortusSophia™** with trademark symbol
- Capitalize UICH (Universal Informational Content Header)
- Be consistent with existing terminology

## Sealed Artifacts

Some documents are marked as "sealed" or authoritative. These require special approval to modify:

- Core UICH specification
- Master metadata schema
- Root path conventions
- Primary trademark guidelines

Propose changes to sealed artifacts through issues first, before submitting PRs.

## Questions?

Open an issue for clarification before starting work on uncertain contributions.

## Code of Conduct

Maintain professional, respectful communication. Focus on governance and technical matters.
